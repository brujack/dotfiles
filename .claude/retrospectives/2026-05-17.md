# Retrospective — 2026-05-17 (dotfiles)

**Period:** 2026-04-17 to 2026-05-17
**Repo(s):** dotfiles
**PRs merged:** 25 (PRs #60–84)
**Direct master commits:** ~few (mostly docs, memory commits)
**Total commits:** 249

## Summary

A heavy 30-day period dominated by **brew/cask drift sophistication**, the **ai-config repo split** (PRs #80–83), and ongoing **plugin/skill management** for Claude Code + Cursor. The repo evolved from "install everything" to "install what this machine actually needs, track drift, and stay in sync with ai-config." 249 commits is the highest of any repo this period.

## What got easier

- **Brew drift detection is now capability-aware** (PRs #60–67) — `_brewfile_parse_inactive` filter means packages installed for inactive profiles don't show as drift. The full-name fix (PR #60) and auto-tap filter (PR #62) eliminated most false-positive drift noise.
- **ai-config split shipped** (PRs #80–83) — `.claude/` and `.cursor/` no longer live in dotfiles; they're managed by the dedicated ai-config repo with symlinks. Cleaner mental model.
- **Cursor parity automated** — terraform-skill installed for both Claude Code and Cursor (PRs #70–73), warp settings symlinked (PR #76), context-mode plugin (PR #77).
- **Weekly digest automated** (PR #78) — `whats-new-anthropic.sh` and `whats-new-claude-code.sh` scheduled.
- **Post-merge cleanup mandatory** (PR #74) — worktree + branch + pull required after every PR.

## Recurring gotchas (action required)

- **Circular symlinks in `.claude/projects/`** (PR #75) — fixed with explicit guard. Worth a memory file if there isn't one already.
- **`ai-config` pull failing on uncommitted changes** (PR #84) — fixed with `--autostash`. Pattern: any auto-pull of a personal repo should use `--autostash`.
