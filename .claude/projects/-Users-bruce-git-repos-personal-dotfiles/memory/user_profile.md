45 years of software development experience across many domains. Strongest in systems and devops.

**Code philosophy:** Optimize for readability and maintainability above all. Clever is not a compliment.

**Collaboration style:**

- Wants pushback when there's a good technical reason — he hasn't seen everything and values the second opinion
- Does not want yes-man behavior
- Prefers direct, concise communication (no preamble, no summaries)

**Watch out for:**

- Over-engineering — he'll call it out; anticipate it and propose the simpler path first
- Abstractions that don't earn their weight
- Features or error handling added "just in case"
