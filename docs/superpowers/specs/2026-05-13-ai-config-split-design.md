# AI Config Split Design

Extract Claude and Cursor configuration from the public dotfiles repo into a
dedicated private repo (`ai-config`) to keep personal AI tooling, memory files,
skills, and standards off a public GitHub repository.

## Context

`~/git-repos/personal/dotfiles` is a public repo. It currently stores all
Claude Code and Cursor configuration under `.claude/` and `.cursor/` and
symlinks them into `~/.claude/` and `~/.cursor/` at setup time. As Claude
config grows to include memory files, personal standards, and detailed
workflows, keeping it public is increasingly undesirable.

## Decisions

- **New repo:** `ai-config` at `~/git-repos/personal/ai-config`
  (`git@github.com:brujack/ai-config`)
- **Cursor moves too:** `.cursor/` relocates alongside `.claude/` — both are
  personal AI tooling and the `sync-agent-guidance` script reads from one to
  generate the other
- **Layout:** Mirror current structure (Approach A) — `.claude/` and `.cursor/`
  subdirectories at the repo root, per-item symlinks into `~/.claude/` and
  `~/.cursor/` as today
- **Bootstrap:** `setup_env.sh` auto-clones `ai-config` if absent before
  running symlink setup; new machine setup remains a single command

`settings.local.json.example` is committed as reference.
`mcp.json` is generated from the template at setup time and gitignored.

## Changes to dotfiles

### New constants (`lib/constants.sh`)

```bash
readonly AI_CONFIG="ai-config"
readonly AI_CONFIG_DIR="${PERSONAL_GITREPOS}/${AI_CONFIG}"
```

### New function (`lib/workflows.sh`)

```bash
setup_ai_config() {
    if [[ ! -d "${AI_CONFIG_DIR}" ]]; then
        log_info "ai-config not found — cloning..."
        git clone git@github.com:brujack/ai-config "${AI_CONFIG_DIR}" || return 1
    fi
}
```

Called at the top of the `setup_user` workflow, before symlink setup. If the
clone fails, `setup_user` aborts — no partial state.

### Symlink loop (`lib/helpers.sh`)

`setup_symlinks` currently reads from `$DOTFILES/.claude/*` and
`$DOTFILES/.cursor/*`. Both loops change source to `$AI_CONFIG_DIR/.claude/*`
and `$AI_CONFIG_DIR/.cursor/*`. Logic is otherwise identical.

### mcp.json generation (`lib/workflows.sh`)

`setup_claude_mcp` changes its template source from
`$DOTFILES/.claude/mcp.json.template` to
`$AI_CONFIG_DIR/.claude/mcp.json.template`. `GITHUB_PAT` continues to come
from dotfiles `config/local.sh` — no change.

### Makefile

Remove `sync-agent-guidance` and `check-agent-guidance` targets. Remove the
`check-agent-guidance` step from CI (`.github/workflows/ci.yml`).

### Removed from dotfiles

- `.claude/` directory and all contents
- `.cursor/` directory and all contents
- `scripts/sync-agent-guidance.sh`

### Doctor check

Add a check to the `doctor` workflow verifying that `$AI_CONFIG_DIR` exists
and is a valid git repo.

## sync-agent-guidance Migration

`scripts/sync-agent-guidance.sh` moves to `ai-config/scripts/`. The script's
`repo_root()` function resolves the directory containing the script — since
both `.claude/` and `.cursor/` now share the `ai-config` root, the script
logic is unchanged. Only the repo it lives in changes.

`ai-config/Makefile` gets `sync-agent-guidance` and `check-agent-guidance`
targets identical to the current dotfiles ones.

The pre-push hook in `ai-config` runs `make check-agent-guidance` to catch
sync drift before a push reaches GitHub.

## Memory Convention Update

`~/.claude/CLAUDE.md` currently states:

> All repos under `~/git-repos/personal/` have their Claude project memories
> stored in `~/git-repos/personal/dotfiles/.claude/projects/` and symlinked
> into `~/.claude/projects/`.

This updates to reference `ai-config` instead of `dotfiles`. Memory files
migrate via `git mv` to preserve history before the dotfiles `.claude/`
directory is removed.

## ai-config CI

Minimal pipeline — config-only repo, nothing to build or test:

- `secret-scan` job: gitleaks against recent commits (advisory, non-blocking)
- `auto-merge` job: merges PR when `secret-scan` passes
- Triggers on `pull_request` only
- Pre-commit hook: `ggshield secret scan pre-commit`
- Pre-push hook: runs `make check-agent-guidance` (no test suite)

## New Machine Bootstrap Flow

1. SSH keys in place (prerequisite — unchanged)
2. Clone dotfiles: `git clone git@github.com:brujack/dotfiles ~/git-repos/personal/dotfiles`
3. Run `./setup_env.sh -t setup_user`
4. `setup_ai_config` detects `ai-config` absent → clones `git@github.com:brujack/ai-config`
5. Symlink loops run against `ai-config/.claude/` and `ai-config/.cursor/`
6. `setup_claude_mcp` generates `~/.claude/mcp.json` from ai-config template + `GITHUB_PAT`

Single command, same as today.

## Out of Scope

- Migrating the `ai-config` repo itself to use `setup_env.sh` for its own
  bootstrap — it is managed by dotfiles
- Moving `config/local.sh` (machine secrets) — stays in dotfiles
- Any changes to how plugins are managed (`~/.claude/plugins/`)
