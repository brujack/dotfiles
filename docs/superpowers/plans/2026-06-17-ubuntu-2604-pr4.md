# Ubuntu 26.04 PR4: ARM64 Support + Version Bumps — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace all hardcoded `amd64`/`x86_64` arch tokens with dynamic variables so download URLs and APT stanzas work on both x86-64 and ARM64 Linux, and bump all stale version pins to latest stable.

**Architecture:** A central `_LINUX_ARCH` variable (added to `lib/constants.sh`) converts `uname -m` kernel naming (`x86_64`/`aarch64`) to Debian/GitHub naming (`amd64`/`arm64`). AWS CLI is the exception — it uses kernel naming directly, so its URLs use `$(uname -m)` inline. APT `[arch=]` stanzas for tools with real ARM64 packages use `$(dpkg --print-architecture)`; tools without ARM64 Linux builds keep `[arch=amd64]` with a comment.

**Tech Stack:** bash, BATS, GNU sed, uname, dpkg

## Global Constraints

- All shell files must pass `bash -n` and `zsh -n` and `shellcheck` (via `make lint`)
- `make test` must exit 0 after every task
- All new tests follow PATH-mock pattern using `tests/mocks/` executables
- Existing passing tests must not regress
- `_LINUX_ARCH` must be defined before any variable that uses it in `lib/constants.sh`
- Vagrant keeps `amd64` (no ARM64 Linux build); VirtualBox keeps `amd64` (no ARM64 Linux); Microsoft Edge keeps `amd64` (no ARM64 Linux)
- TDD: write failing test first, confirm red, implement, confirm green, commit
