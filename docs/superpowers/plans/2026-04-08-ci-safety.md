### Task 1: Fix auto-merge condition

**Files:**
- Modify: `.github/workflows/ci.yml`

- [ ] **Step 1: Read the current CI file**

Read `.github/workflows/ci.yml` in full to understand the current state before editing.

- [ ] **Step 2: Replace the auto-merge `if` condition**

The current condition (line 28):

```yaml
    if: github.event_name == 'pull_request' && github.actor == 'dependabot[bot]' || github.event_name == 'push'
```

Due to operator precedence this evaluates as `(PR AND dependabot) OR push`. It fires on every push. The intent is to run when there's a push or PR — relying on the `gh pr list` check in the step body to only merge when a PR actually exists.

Replace with an explicit, parenthesized form:

```yaml
    if: github.event_name == 'pull_request' || github.event_name == 'push'
```

### Task 2: Add `lint-macos` job

**Files:**
- Modify: `.github/workflows/ci.yml`

- [ ] **Step 1: Add `lint-macos` job to `.github/workflows/ci.yml`**

Add the following job after the `test` job and before the `auto-merge` job. It runs in parallel with `test` (no `needs` dependency), on `macos-latest`, requiring no additional tool installs:

```yaml
  lint-macos:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v5

      - name: Syntax check all shell scripts (bash)
        run: |
          find . -name '*.sh' \
            -not -path './.worktrees/*' \
            -not -path './tests/mocks/*' \
            | xargs -I{} bash -n {}

      - name: Syntax check all shell scripts (zsh)
        run: |
          find . -name '*.sh' \
            -not -path './.worktrees/*' \
            -not -path './tests/mocks/*' \
            | xargs -I{} zsh -n {}
```

Note: `lint-macos` is **not** added to the `auto-merge` job's `needs`. It is advisory — it surfaces macOS-specific syntax drift without blocking merges until confidence is established.
