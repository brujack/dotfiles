# Claude Code — What's New (2026-07-20)

## New Features

- **EndConversation tool**: Claude can now end sessions with highly abusive users or jailbreak attempts, consistent with claude.ai behavior since 2025.
- **Screen reader mode**: Opt-in plain-text rendering via `--ax-screen-reader`, `CLAUDE_AX_SCREEN_READER=1`, or `"axScreenReader": true` in settings.
- **`/fork` redesigned as a background branch**: `/fork` now copies your conversation into a new background agent row in `claude agents`; use `/subtask` for the previous in-session subagent behavior.
- **Subagent and WebSearch loop guards**: New per-session caps (default 200 each) on subagent spawns and WebSearch calls prevent runaway loops; `/clear` resets the budget.
- **MCP calls auto-background after 2 minutes**: Long-running MCP tool calls move to the background automatically so the session stays responsive; configurable via `CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS`.
- **`claude auto-mode reset`**: Restores default auto-mode configuration with a confirmation prompt (pass `--yes` to skip).
- **Live elapsed-time counter on collapsed tool summaries**: Long-running tool calls now visibly tick instead of looking stuck.
- **`--forward-subagent-text` flag**: New flag and `CLAUDE_CODE_FORWARD_SUBAGENT_TEXT` env var include subagent text and thinking in stream-json output.
- **`vimInsertModeRemaps` setting**: Map two-key vim insert-mode sequences (e.g. `jj → Esc`).
- **`CLAUDE_CODE_PROCESS_WRAPPER`**: Routes every Claude Code self-spawn through a required corporate launcher executable.
- **`/resume` in agent view opens a full session picker**: Includes previously-deleted sessions, resuming the chosen one as a background session.
- **`/fork` renamed; `/subtask` is the new in-session delegation command**.

## Improvements

- **Bash and PowerShell permission checks hardened**: Multiple security bypass paths fixed — PowerShell 5.1 sessions, file-descriptor redirects, commands over 10,000 characters, zsh variable subscripts, and certain `help`/`man` commands now correctly require approval.
- **`docker` commands now prompt for permission**: Commands using daemon-redirect flags (`--url`, `--connection`, `--identity`, Podman remote mode) no longer run silently.
- **Agent tool hardened against indirect prompt injection**: Content read by a subagent can no longer redirect the parent agent.
- **Auto mode permission classifier defaults to Sonnet 5**: Applied to external sessions, validated on first request and pinned for the session.
- **Memory leaks fixed in long sessions**: MCP stdio server stderr accumulation (up to 64 MB/server), unbounded LSP document cache (now LRU-capped at 50), and large tool-result payload growth are all resolved.
- **Session transcript size reduced up to 79x** in edit-heavy sessions by pruning superseded file-history backups.
- **`/verify` and `/code-review` no longer auto-invoke**: Claude only runs these skills when you explicitly type `/verify` or `/code-review`.
- **ISO `modified` timestamp added to memory file frontmatter**.
- **`message.uuid`, `client_request_id`, and `tool_source` added to OpenTelemetry log events** for message-level correlation and tool provenance.
- **`CLAUDE_CODE_OTEL_CONTENT_MAX_LENGTH`**: New env var to configure the 60 KB truncation limit on OTel content attributes.
- **Reasoning effort added to `subagentStatusLine` payload**: Custom agent rows can now render model and effort together.
- **`/doctor` adds startup warning** for `Write(path)`, `NotebookEdit(path)`, and `Glob(path)` permission rules — use `Edit(path)` or `Read(path)` instead.
- **Bedrock, Vertex, and Claude Platform on AWS default to Claude Opus 4.8**.
- **`auto mode` no longer reads `autoMode` from `.claude/settings.local.json`**: Use `~/.claude/settings.json` instead.

## Bug Fixes

- Fixed a permission-check bypass affecting Windows PowerShell 5.1 sessions.
- Fixed `isolation: 'worktree'` subagents able to run git-mutating commands against the main repo checkout.
- Fixed the `ultracode` keyword opt-in firing on non-human-originated input such as webhook payloads and relayed PR comments.
- Fixed plugins loaded via `--settings` CLI flag not loading (regression since v2.1.181).
- Fixed background sessions silently stopping mid-turn after sleep/wake cycles.
- Fixed background agents killed by the user auto-respawning on restart.
- Fixed `/clear` not resetting the session cost counter.
- Fixed `claude update` and `claude doctor` hanging silently when a shell-config path is a directory.
- Fixed memory frontmatter values silently truncated at an inline `#` character.
- Fixed Bedrock auth failing with "Session token not found" when `sso_region` differs from the Bedrock region (2.1.207 regression).
- Fixed parallel Claude Code sessions all logging out simultaneously after sleep/wake.
- Fixed `/loop` hiding the session from `/resume` after a single use.
- Fixed Edit tool failing on files modified after reading when the target text still matches uniquely.
- Fixed feature flags going stale in long-running sessions after OAuth token rotation.
- Fixed scheduled tasks refusing their own configured prompt as untrusted input.
- Fixed PowerShell tool commands hanging until timeout when a child process waited on stdin (Windows).
- Fixed stream-json output truncation when piping large responses from `claude -p`.
- Fixed Bedrock streaming requests failing with a misleading "Truncated event message received" behind a gateway.
- Fixed `/release-notes` injecting the full changelog into model context on "Show all".
- Fixed multiple memory leaks: pasted image retention in agent view, MCP stdio server stderr, LSP doc accumulation, and large tool-result payloads in headless sessions.
- Fixed `SendMessage` silently misrouting when a re-spawned agent reuses a previous agent's name.
- Fixed plan mode auto-running file-modifying Bash commands (e.g. `touch`, `rm`) without a permission prompt.
- Fixed `/ultrareview` refusing to run in repos with no merge base.
