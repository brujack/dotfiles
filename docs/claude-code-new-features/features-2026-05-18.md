# Claude Code — What's New (2026-05-18)

## New Features

- **Agent view** (`claude agents`): a unified list of all Claude Code sessions — running, blocked, or done — now in Research Preview.
- **`/goal` command**: set a completion condition and Claude works across turns until it is met, showing live elapsed/turns/tokens in an overlay panel.
- **Plugin dependency enforcement**: `claude plugin disable` refuses if another enabled plugin depends on the target and offers a copy-pasteable disable chain; `claude plugin enable` force-enables transitive dependencies.
- **`claude agents` configuration flags**: new `--add-dir`, `--settings`, `--mcp-config`, `--plugin-dir`, `--permission-mode`, `--model`, `--effort`, and `--dangerously-skip-permissions` flags scope dispatched background sessions.
- **`worktree.bgIsolation: "none"` setting**: lets background sessions edit the working copy directly without `EnterWorktree`, for repos where worktrees are impractical.
- **`ANTHROPIC_WORKSPACE_ID` env var**: scopes workload identity federation tokens to a specific workspace.
- **Hook `args: string[]` exec form**: spawns hook commands without a shell so path placeholders never need quoting; `continueOnBlock` option feeds rejection reason back to Claude and continues the turn.
- **`/scroll-speed` command**: tunes mouse wheel scroll speed with a live preview.
- **`claude plugin details <name>`**: shows a plugin’s component inventory and projected per-session token cost.
- **Transcript view navigation**: `?` for shortcuts, `{`/`}` to jump between user prompts, `v` to toggle shortcut panel.
- **Rewind menu “Summarize up to here”**: compresses earlier context while keeping recent turns intact.
- **Projected context cost** shown in the `/plugin` marketplace browse pane (per-turn and per-invocation token estimates).

## Improvements

- Fast mode now uses Opus 4.7 by default (was Opus 4.6); pin to 4.6 with `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1`.
- Background sessions preserve the model and effort level set before going idle.
- PowerShell tool now passes `-ExecutionPolicy Bypass` by default and is enabled by default on Windows for Bedrock, Vertex, and Foundry users.
- `/bg` and `←`-detach preserve `--mcp-config`, `--settings`, `--fallback-model`, `--add-dir`, `--plugin-dir`, and `--strict-mcp-config` across respawn.
- Plugins with a root-level `SKILL.md` and no `skills/` subdirectory are now surfaced as a skill.
- Spinner warms to amber after 10 seconds of thinking to signal Claude is still working.
- `/mcp` Reconnect picks up `.mcp.json` edits without a restart.
- Agent tool `subagent_type` matching now accepts case- and separator-insensitive values.

## Bug Fixes

- Fixed corrupt `.credentials.json` with a non-array `scopes` value hanging the CLI on startup.
- Fixed stop hooks that block repeatedly looping forever — the turn now ends with a warning after 8 consecutive blocks.
- Fixed background agents false-positive worker-stall detection after host sleep or macOS App Nap.
- Fixed background sessions disappearing and daemon reconnect failing after macOS sleep/wake.
- Fixed `MCP_TOOL_TIMEOUT` not raising the per-request fetch timeout for remote HTTP/SSE MCP servers.
- Fixed background side-queries using an unavailable Haiku model ID on Bedrock/Vertex/Foundry.
- Fixed `/goal` silently hanging when `disableAllHooks` is set.
- Fixed settings hot-reload not detecting edits to symlinked `~/.claude/settings.json`.
- Fixed unbounded memory growth when an HTTP/SSE MCP server streamed non-protocol data.
- Fixed `autoAllowBashIfSandboxed` not auto-approving commands with shell expansions.
- Fixed `Skill(name *)` permission rules — wildcard form now works as a prefix match.
