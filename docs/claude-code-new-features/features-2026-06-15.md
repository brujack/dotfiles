# Claude Code — What's New (2026-06-15)

## New Features
- **Claude Fable 5** is now available — a Mythos-class model with capabilities exceeding any previously generally-available model; update to 2.1.170+ for access
- **Sub-agents can now spawn sub-agents** up to 5 levels deep, enabling deeper multi-agent workflows
- **`--safe-mode` flag** (and `CLAUDE_CODE_SAFE_MODE` env var) starts Claude Code with all customizations disabled (CLAUDE.md, plugins, skills, hooks, MCP servers) for troubleshooting
- **`/cd` command** lets you move a session to a new working directory without breaking the prompt cache mid-session
- **Self-hosted runner `post-session` hook** runs after a session ends and before workspace deletion, so you can snapshot uncommitted work or export logs
- **`enforceAvailableModels` managed setting** — when enabled, constrains the Default model to the `availableModels` allowlist and prevents user/project settings from widening it
- **Session titles generated in your conversation language** — set the `language` setting to pin a specific language
- **`footerLinksRegexes` setting** lets you add regex-matched link badges to the footer row via user or managed settings
- **`disableBundledSkills` setting** hides bundled skills, workflows, and built-in slash commands from the model
- **`wheelScrollAccelerationEnabled` setting** disables mouse-wheel scroll acceleration in fullscreen mode
- **Plugin marketplace search bar** in `/plugin` for browsing available plugins
- **Amazon Bedrock** now reads the AWS region from `~/.aws` config files when `AWS_REGION` isn't set, matching AWS SDK precedence
- **[VSCode]** Usage attribution added to the Account & usage dialog (`/usage`) showing cache misses, long context, subagents, and per-skill/agent/plugin/MCP breakdowns

## Improvements
- Bedrock credential caching now lasts until token expiration instead of a fixed 1 hour
- `claude update` announces the target version before downloading
- Long-conversation performance improved by removing redundant message normalization
- Reduced idle CPU usage: `/goal` status chip no longer re-renders at 5 Hz while idle
- Skill hot-reload now only re-announces changed skills instead of the full listing
- `/workflows` opens immediately even while a turn is in progress
- The "CLAUDE.md is too long" warning threshold now scales with the model's context window
- Restored 5-minute idle timeout on Vertex/Foundry to abort stalled streams instead of hanging

## Bug Fixes
- Fixed hook `if` conditions for Read/Edit/Write tool paths — patterns like `Edit(src/**)` and `Read(.env)` now match correctly
- Fixed Linux sandbox failing to start when `.claude/settings.json` is a symlink with an absolute target
- Fixed `/copy` and mouse-selection copy not reaching the system clipboard inside tmux over SSH
- Fixed Remote Control silently switching the session's model when connecting from web/mobile
- Fixed auto mode failing on Fable 5 for organizations without Opus 4.8 enabled
- Fixed `availableModels` restrictions not being applied to subagent model overrides and agent dispatch picker
- Fixed `WebFetch(domain:*.example.com)` wildcard domain rules never matching subdomains
- Fixed background agents ignoring project-level `env` values when dispatched onto pre-warmed workers
- Fixed sessions using 1M context without usage credits getting permanently stuck
- Fixed `claude -p` appearing to hang on Windows while waiting for the slash-command/skill scan
- [VSCode] Fixed PowerShell tool calls rendering as raw JSON instead of a proper command display
