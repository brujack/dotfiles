# Claude Code — What's New (2026-05-08)

## New Features

- `settings.autoMode.hard_deny` allows defining auto-mode classifier rules that block actions unconditionally, regardless of user intent or allow exceptions.
- `CLAUDE_CODE_ENABLE_FEEDBACK_SURVEY_FOR_OTEL` re-enables the session quality survey for enterprises capturing responses via OpenTelemetry.
- `worktree.baseRef` setting (`fresh` | `head`) controls whether `--worktree` and `EnterWorktree` branch from `origin/<default>` or local HEAD.
- Hooks now receive the active effort level via the `effort.level` JSON input field and `$CLAUDE_EFFORT` environment variable.
- `CLAUDE_CODE_SESSION_ID` is now injected into Bash tool subprocess environments, matching the `session_id` passed to hooks.
- `CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1` opts out of the fullscreen alternate-screen renderer, keeping conversation in native terminal scrollback.
- `--plugin-url <url>` flag fetches a plugin `.zip` archive from a URL for the current session.
- `CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE` enables background upgrades via Homebrew or WinGet with a restart prompt.
- `claude project purge [path]` deletes all Claude Code state for a project, with `--dry-run`, `--yes`, and `--interactive` options.
- `claude ultrareview [target]` runs `/ultrareview` non-interactively from CI or scripts, printing findings to stdout.
- `claude auth login` now accepts an OAuth code pasted into the terminal when the browser callback cannot reach localhost (WSL2, SSH, containers).

## Improvements

- WSL2: image paste from the Windows clipboard now works via a PowerShell fallback when xclip/wl-paste cannot read image data.
- Visual consistency improved across slash command dialogs — standardized footer hints, spacing, arrow-key styling, and immediate frame rendering during loading.
- `/model` picker now lists models from your gateway's `/v1/models` endpoint when using an Anthropic-compatible gateway.
- Ctrl+R history picker now defaults to searching all prompts across all projects (matching pre-2.1.124 behavior); press Ctrl+S to narrow scope.
- MCP servers that hit a transient error during startup now auto-retry up to 3 times instead of staying disconnected.
- Memory usage improved by releasing warm-spare background workers under memory pressure.

## Bug Fixes

- Fixed MCP servers in `.mcp.json`, plugins, and claude.ai connectors silently disappearing after `/clear` in VS Code, JetBrains, and Agent SDK.
- Fixed MCP OAuth refresh tokens being lost when multiple servers refresh concurrently — daily re-authentication no longer required.
- Fixed `--resume`/`--continue` not finding sessions when the project path contains underscores.
- Fixed parallel sessions all failing with 401 after a refresh-token race wiped shared credentials.
- Fixed plan mode not blocking file writes when a matching `Edit(...)` allow rule exists.
- Fixed unbounded memory growth (10GB+ RSS) when a stdio MCP server writes non-protocol data to stdout.
- Fixed `Bash(mkdir *)`, `Bash(touch *)`, and similar allow rules not being honored for in-project paths.
