# Claude Code — What's New (2026-07-06)

## New Features
- **Claude Sonnet 5** is now the default model with a 1M-token context window and promotional pricing ($2/$10/Mtok through August 31)
- Organization default models: admins set the default in the org console; shown as "Org default" in `/model`
- `/dataviz` skill for chart and dashboard design guidance with a runnable color-palette validator
- Claude in Chrome is now generally available
- Background agent notifications: the `Notification` hook fires when sessions need input (`agent_needs_input`) or finish (`agent_completed`)
- Background agents auto-commit, push, and open a draft PR when they finish code work in a worktree
- Subagents now run in the background by default, so Claude keeps working while they run
- Stacked slash-skill invocations (e.g. `/skill-a /skill-b do XYZ`) now load all leading skills (up to 5)
- Clickable file attachments in chat — Cmd/Ctrl-click reveals the file in Finder/Explorer
- `AskUserQuestion` dialogs no longer auto-continue by default; opt into an idle timeout via `/config`

## Improvements
- The default permission mode is now labeled "Manual" across CLI, `--help`, VS Code, and JetBrains
- Subagents and context compaction now inherit the session's extended thinking configuration for better output quality
- The built-in Explore agent inherits the main session's model (capped at Opus) instead of running on Haiku
- `/code-review` consolidated five cleanup finders into one, cutting token usage by ~25%
- Transient server 429 rate-limit errors now retry automatically with backoff for subscribers
- Streaming idle watchdog is on by default for all providers — aborts and retries after 5 min of silence
- Background sessions survive process restarts including on Windows via shell handoff
- API retry UX: error reason shown after the second attempt; status page link when the API is overloaded
- Screen-reader output improved: decorative glyphs suppressed, transcript symbols read as short labels

## Bug Fixes
- Fixed background agents stuck after a crash left a stale daemon lock file whose PID the OS had reused
- Fixed background sessions silently stopping mid-turn after sleep/wake
- Fixed SSL certificate errors burning retries — they now fail immediately with the actionable fix hint
- Fixed the background-agent daemon on Linux self-terminating every ~50 seconds after an unclean shutdown
- Fixed subagents reporting API errors (rate limits, usage limits) as successful results to the parent
- Fixed waking a background job permanently deleting its conversation history
- Fixed MCP OAuth requesting the full `scopes_supported` catalog, causing failures on GitLab and enterprise IdPs
- Fixed PowerShell `git diff`, `egrep`/`fgrep`, and `|`-quoted search patterns falsely reported as failures
