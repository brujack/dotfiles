# Claude Code — What's New (2026-06-08)

## New Features

- **Fallback model support**: Configure up to three fallback models with the `fallbackModel` setting; Claude Code retries once on the fallback when the API rejects an unexpected non-retryable error.
- **Thinking controls**: `MAX_THINKING_TOKENS=0`, `--thinking disabled`, and the per-model toggle now disable thinking on models that think by default via the Claude API.
- **Managed version gates**: New `requiredMinimumVersion` / `requiredMaximumVersion` managed settings let admins lock Claude Code to an approved version range.
- **Plugin list command**: `/plugin list` shows all installed plugins with optional `--enabled`/`--disabled` filters.
- **Hook feedback loop**: Stop and SubagentStop hooks can now return `hookSpecificOutput.additionalContext` to give Claude feedback and keep the turn going.
- **Glob deny rules**: Deny rules now accept `"*"` in the tool-name position to block all tools.
- **Shell startup file prompts**: Claude Code now prompts before writing to `.zshenv`, `.zlogin`, `.bash_login`, and `~/.config/git/` to prevent unintended command execution.
- **Build-tool config safety**: `acceptEdits` mode prompts before writing files that grant code execution (`.npmrc`, `.yarnrc*`, `.bazelrc`, `.devcontainer/`, etc.).
- **`claude agents --json` `waitingFor`**: The JSON output now includes what a waiting session is blocked on (e.g., a permission prompt).
- **Parallel tool resilience**: A failed Bash command in a parallel batch no longer cancels other calls in the same batch.
- **`/btw` copy shortcut**: Press `c` in a `/btw` answer to copy the raw markdown to the clipboard.

## Improvements

- `claude update` announces the target version before downloading instead of going silent.
- `claude agents` rows show a `done/total` count when work is fanned out across agents.
- `/mcp` collapses claude.ai connectors you've never signed in to behind a "Show unused connectors" row.
- Edit no longer requires a separate Read after viewing a file with `grep` — single-file grep commands now satisfy the read-before-edit check.
- Remote Control now shows as a persistent footer pill with a session link instead of a startup message.
- Improved auto mode classifier latency by reducing reasoning on routine actions.
- Renamed the dynamic-workflow trigger keyword from `workflow` to `ultracode`.
- `OTEL_RESOURCE_ATTRIBUTES` values are included as labels on metric datapoints for custom dimension slicing.

## Bug Fixes

- Fixed JetBrains IDE terminal flickering on 2026.1+ by enabling synchronized output.
- Fixed Shift+non-ASCII characters (e.g., Shift+ä → Ä) being dropped in Kitty keyboard protocol terminals.
- Fixed `claude -p` hanging forever after its final result when a backgrounded command never exits.
- Fixed bash commands failing under bazel and EDR-protected Go workflows due to `$TMPDIR` being incorrectly overridden.
- Fixed `/doctor` showing a contradictory "Not inside a remote session" failure when run inside a remote session.
- Fixed `claude mcp` list/get/add printing secrets to the terminal; `${VAR}` references are no longer expanded.
- Fixed background sessions re-attached after overnight retire re-running the original prompt.
- Fixed voice mode failing to connect when the project directory or branch name contains non-ASCII characters.
- Fixed Windows permission rules never matching when spelled with backslashes or case-variant paths.
