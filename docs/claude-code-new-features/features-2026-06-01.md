# Claude Code — What's New (2026-06-01)

## New Features

- **Auto mode on Bedrock, Vertex, and Foundry**: Opus 4.7 and 4.8 auto mode is now available on cloud provider deployments — opt in with `CLAUDE_CODE_ENABLE_AUTO_MODE=1`.
- **Opus 4.8 is here**: Now defaults to high effort; fast mode runs at 2x the standard rate for 2.5x the speed.
- **Dynamic workflows**: Ask Claude to orchestrate work across tens to hundreds of background agents; run `/workflows` to view your runs.
- **Plugins auto-load from `.claude/skills`**: No marketplace required — directories are scanned automatically, and `claude plugin init <name>` scaffolds new plugins in place.
- **`/code-review --fix`**: Applies review findings (reuse, simplification, efficiency) directly to your working tree; `/simplify` now invokes this instead of the full bug-hunting review.
- **`MessageDisplay` hook**: New hook event lets hooks transform or hide assistant message text as it is displayed.
- **`SessionStart` hooks extended**: Hooks can now return `reloadSkills: true` to rescan skill directories, and set the session title via `hookSpecificOutput.sessionTitle`.
- **Skills can restrict tools**: Frontmatter `disallowed-tools` removes specific tools from the model while a skill is active.
- **`/reload-skills`**: Re-scans skill directories without restarting the session.
- **Shell commands as background sessions**: In `claude agents`, type `! <command>` to run a shell command as an attachable background session.
- **Auto mode no longer requires opt-in consent**: Previously required an explicit consent step.

## Improvements

- **`/model` saves as default**: Selecting a model now persists it as the default for new sessions; press `s` to switch for the current session only.
- **Lean system prompt is now the default**: Applies to all models except Haiku, Sonnet, and Opus 4.7 and earlier.
- **Status line scripts receive terminal dimensions**: `COLUMNS` and `LINES` are now set so scripts can size output to the terminal width.
- **Streaming tool execution always on**: No longer behind a feature flag on Bedrock, Vertex, Foundry, or with telemetry disabled.
- **Opus 4.8 support in `/claude-api` skill**: Includes migration guidance from 4.7 → 4.8.
- **Vim mode reverse history search**: `/` in NORMAL mode now opens reverse history search, matching bash/zsh vi-mode behavior.
- **Thinking summaries improved**: Stay readable for at least 3 seconds, render as markdown, and cap at 10 lines.
- **`claude agents` PR column**: Shows `PR #N` for a single PR or `N PRs` for multiple.
- **`/terminal-setup` disables GPU acceleration**: Prevents garbled-text rendering in VS Code, Cursor, and Windsurf integrated terminals.

## Bug Fixes

- Fixed corrupt or zero-byte images crashing the request instead of becoming a text placeholder.
- Fixed sandbox network permission prompts appearing in auto/bypass-permissions mode on desktop and IDE extensions.
- Fixed background agent worktrees being orphaned after the 30-day job retention sweep.
- Fixed background sessions re-attached after sleep/wake reporting the wrong date to the model.
- Fixed copy-on-select not reaching the system clipboard inside tmux (regression in 2.1.153).
- Fixed `--resume` not reporting background subagents that were running when Claude Code previously exited.
- Fixed thinking blocks being modified when using Opus 4.8, leading to API errors.
- Fixed `rm -rf $HOME` not being blocked when `HOME` has a trailing slash.
- Fixed Windows update failures showing a generic error instead of a clear "close other sessions" message.
- Fixed stateful MCP servers without optional GET SSE stream reconnect-looping on `tools/list`.
- Fixed subagent frontmatter MCP servers ignoring `--strict-mcp-config` and enterprise managed MCP config.
- Fixed excessive memory usage when resuming a session by transcript path on machines with many stored sessions.
- Fixed `EnterWorktree` not being available immediately in background sessions.
