# Claude Code — What's New (2026-07-13)

## New Features

- Auto mode is now available on Bedrock, Vertex AI, and Foundry without the `CLAUDE_CODE_ENABLE_AUTO_MODE` opt-in env var; disable via `disableAutoMode` in settings.
- Bedrock, Vertex, and Claude Platform on AWS now default to Claude Opus 4.8 instead of Sonnet.
- `/doctor` is now a full interactive setup checkup that can diagnose and fix issues; `/checkup` is an alias.
- A warning now appears when your login is about to expire, letting you re-authenticate before background sessions are interrupted.
- A grey ⏸ badge now appears in the footer when manual permission mode is active, making the current mode always visible.
- `/cd` now shows directory path suggestions, matching the behavior of `/add-dir`.
- A "Dynamic workflow size" setting in `/config` controls how large Claude generally makes dynamic workflows (small/medium/large).
- Added `workflow.run_id` and `workflow.name` OpenTelemetry attributes to telemetry emitted by workflow-spawned agents.
- Background agents now upgrade to a new version in the background right after a Claude Code update, avoiding a slow stale-session upgrade on attach.
- `/commit-push-pr` now auto-allows `git push` to the repo's configured push remote (`remote.pushDefault` or the sole remote) in addition to `origin`.
- A new `/doctor` check proposes trimming oversized checked-in `CLAUDE.md` files Claude could derive from the codebase.
- Background task notifications now explicitly state no human input has occurred, preventing fabricated in-transcript approvals from being acted on.

## Improvements

- Auto-update binary downloads now stream to disk instead of buffering in memory, cutting the updater's peak memory usage by ~400 MB.
- Binary size is reduced by ~7 MB and startup memory by ~7 MB by lazy-loading a large bundled dependency.
- `/code-review` findings quality improved on `claude-opus-4-8` across all effort levels.
- Agent view rows now display a colored state word and a classifier-written headline instead of raw tool call text.
- Shell-injection fix: `${user_config.*}` in plugin hook shell-form commands is now rejected; use exec form (`args` array) or `$CLAUDE_PLUGIN_OPTION_<KEY>` instead.
- Plugin option values (`pluginConfigs`) are no longer read from project-level `.claude/settings.json`; only user, `--settings`, and managed settings are honored.

## Bug Fixes

- Fixed terminal freezing and keystrokes lagging while streaming responses containing very long lists, tables, or code blocks.
- Fixed remote managed settings from non-interactive runs (`claude -p`, SDK) being permanently recorded as consented without showing the security consent dialog.
- Fixed spurious prompt-injection warnings triggered by benign system-generated conversation updates.
- Fixed the auto-updater overwriting a custom launcher script or symlink at `~/.local/bin/claude` on every release; `/doctor` now reports an externally managed launcher.
- Fixed background sessions that entered a git worktree resuming blank after a cold reopen from the agent list.
- Fixed Remote Control task status updates being lost when the connection recovered from a network interruption or credential refresh.
- Fixed Bedrock repeatedly requesting fresh AWS SSO credentials from IAM Identity Center on every API request.
- Fixed background agents inheriting a stale `PATH` from the daemon instead of the dispatching shell on Windows.
- Fixed a per-turn CPU and memory regression where the context-usage indicator re-analyzed the entire transcript after every turn.
- Fixed worktree-isolated subagents sometimes running shell commands in the parent checkout instead of their own worktree.
