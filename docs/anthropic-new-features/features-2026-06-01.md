# Anthropic & Claude API — What's New (2026-06-01)

## Model & API Changes

- Claude Opus 4.8 launched with a 1M token context window, 128k max output tokens, adaptive thinking (triggers reasoning only when needed), and high-resolution image input.
- Mid-conversation system messages now supported on Opus 4.8 — send `role: "system"` at non-first positions to preserve cache hits as instructions change, no beta header needed.
- Messages API returns refusal categories in `stop_details` on Opus 4.8, enabling downstream routing on declined requests.
- Setting `temperature`, `top_p`, or `top_k` to a non-default value returns a 400 error on Opus 4.8 (same behavior as Opus 4.7).
- Fast mode for Opus 4.8 available as a research preview (API only); fast mode for Opus 4.6 is deprecated with removal ~30 days post-launch.
- Claude Platform on AWS now supports Managed Agents, webhooks, multiagent orchestration, and self-hosted sandboxes, with new IAM actions.
- Cache diagnostics in public beta: pass `diagnostics.previous_message_id` to receive a `cache_miss_reason` in the response (requires `cache-diagnosis-2026-04-07` beta header).
- MCP tunnels available as a research preview for connecting to MCP servers in private networks.
- Managed Agents now support mid-session MCP server/tool config updates; tool outputs exceeding 100K tokens auto-spill to a sandbox file with a truncated preview sent to the model.
- Rate Limits API launched, allowing admins to query organization and workspace rate limits programmatically.
- Advisor tool in public beta: pair a fast executor model with a higher-intelligence advisor for agentic workloads at lower cost.
- Automatic caching for the Messages API: add a single `cache_control` field and the system moves the cache point forward automatically as conversations grow.
- Web search, programmatic tool calling, web fetch, and code execution tools are all generally available (no beta header required); web search now supports dynamic filtering via code execution.
- Structured outputs are GA for Sonnet 4.5, Opus 4.5, and Haiku 4.5; `output_format` has moved to `output_config.format`.
- 1M token context window beta retired for Sonnet 4.5 and Sonnet 4 — requests over 200k tokens now return an error; migrate to Sonnet 4.6 or Opus 4.6 where it's GA.
- Models API (`GET /v1/models`) now returns `max_input_tokens`, `max_tokens`, and a `capabilities` object per model.
- Claude Sonnet 4 and Opus 4 deprecated, retiring June 15, 2026.

## SDK Changes

- Python SDK added support for `claude-opus-4-8`, mid-conversation system blocks, and `usage.output_tokens_details`.
- Python SDK added support for custom file size caps in the Files API.
