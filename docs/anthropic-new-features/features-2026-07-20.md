# Anthropic & Claude API — What's New (2026-07-20)

## Model & API Changes
- Claude Sonnet 5 launched with a 1M token context window, 128k max output tokens, adaptive thinking on by default (manual extended thinking removed), restricted sampling parameters, and a new tokenizer producing ~30% more tokens for the same text.
- Claude Opus 4.8 launched as the most capable generally available model, with 1M context by default, effort defaulting to high, lower minimum cacheable prompt length, and a fast mode research preview.
- Claude Fable 5 and Claude Mythos 5 launched with 1M context, always-on adaptive thinking, safety-classifier refusals (with an opt-in fallback-model parameter), and a new tokenizer (~30% more tokens).
- Claude Platform on AWS launched, bringing the full Messages API, Files API, Managed Agents, and tool use to AWS-native endpoints with AWS billing/IAM.
- Mid-conversation system messages are now supported on Fable 5, Mythos 5, and Opus 4.8 without a beta header, preserving prompt cache hits when instructions change mid-session.
- The Admin API now supports managing Claude Enterprise org members, groups, and custom roles in beta.
- API keys and Admin API keys can now be created with an expiration date/preset.
- Rate limits were raised across the API and usage tiers consolidated into three (Start, Build, Scale).
- Fast mode was removed for Opus 4.6 and deprecated for Opus 4.7 (removal July 24).
- Claude Sonnet 4 and Opus 4 models were retired; Opus 4.1 deprecation was announced (retiring August 5).
- Code execution tool gained a version with REPL state persistence (required for programmatic tool calling) and a version disclosing the 90-second per-cell execution limit.
- Web search and web fetch tools added a `response_inclusion` parameter to drop consumed result blocks from responses.
- A new `agent-memory-2026-07-22` beta header changes memory-store listing behavior (stable ordering, restricted `depth` values, exact-segment `path_prefix` matching).
- Managed Agents gained event-delta streaming, backward session pagination, per-session config overrides, vault credential injection controls, scheduled deployments, self-hosted sandboxes, and lifecycle webhooks; multiagent orchestration and Outcomes reached public beta.
- Refusal responses now return `stop_reason: "refusal"` with a documented `stop_details` category, and are not billed when no output was generated.
- Usage responses now report `thinking_tokens` within `output_tokens_details`.

## SDK Changes
- The Python SDK added support for Dreams and MCP Tunnels, and now sends the `agent-memory-2026-07-22` header on memory store calls in place of the prior header.
- Python, TypeScript, Go, Java, Ruby, PHP, and C# SDKs added support for the persistent-REPL code execution tool version.

## Bug Fixes
- The Python SDK now keeps credential material out of traceback frame locals by wrapping it in `SecretStr`.
