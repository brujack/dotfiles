# Anthropic & Claude API — What's New (2026-05-25)

## Model & API Changes

- **claude-opus-4-7 added** — new model variant with token budget support and user profiles
- **CMA Memory reached public beta** — Claude Managed Agents memory is now publicly available
- **Managed Agents expanded** — multiagent orchestration, outcome reporting, webhooks, and vault validation added; APIs further improved in a follow-up release
- **Self-hosted sandboxes in CMA** — sandbox helper support added for Claude Managed Agents
- **BetaManagedAgentsSearchResultBlock types added** — new type definitions for managed agent search results
- **Cache diagnostics beta** — new beta flag to enable cache diagnostic metadata in responses
- **AWS client for Claude Platform** — dedicated client for accessing Claude Platform on AWS
- **OIDC workspace targeting** — token exchange can now target a specific workspace for OIDC federation

## SDK Changes

- **Workload Identity Federation** — new auth method alongside interactive OAuth and named auth profiles
- **Header setting via env vars** — request headers can now be configured through environment variables
- **thinking-token-count beta** — streaming now reports estimated token counts inside thinking block deltas when the beta is enabled
- **Pydantic iterator eager validation** — pydantic iterators are now validated eagerly for faster error surfacing
- **AWS multi-region Vertex fix** — async Vertex client now correctly handles `us`/`eu` multi-region `base_url` branches

## Bug Fixes

- Streaming accumulator now carries `encrypted_content` through beta compaction correctly
- `SessionToolRunner` skips tool calls it does not own, preventing incorrect dispatch
- `stop_details` from `message_delta` events is now propagated onto the accumulated `Message`
- Multipart file arrays use the correct field name format
- Missing `f`-string prefix in file type error message corrected
