## Model & API Changes

- **Claude Fable 5** (`claude-fable-5`) launched with 1M context, 128k max output, and always-on adaptive thinking; `thinking: {"type": "disabled"}` returns a 400.
- Fable 5 and Mythos 5 use a new tokenizer that produces ~30% more tokens than pre-Opus-4.7 models â€” use the token counting API to re-measure prompts.
- Fable 5 returns `stop_reason: "refusal"` when classifiers block a request; refused requests with no output generated are not billed.
- An opt-in `fallbacks` parameter (beta) re-runs refused Fable 5 requests on another model automatically.
- **Claude Opus 4.8** launched with 1M context, 128k max output, adaptive thinking that triggers only when needed, and a lowered prompt-cache minimum of 1,024 tokens.
- Mid-conversation system messages are now supported on Opus 4.8 â€” send `role: "system"` after a user turn without breaking cache hits.
- Code execution tool version `code_execution_20260521` discloses the 90-second per-cell time limit in the tool description; no beta header required.
- Web search and web fetch tools now accept a `response_inclusion` parameter to drop consumed result blocks from the API response, reducing context overhead in agentic loops.
- The advisor tool now accepts `max_tokens` to cap advisor output per call, cutting latency and cost for workloads that don't need full-length responses.
- Claude Managed Agents adds scheduled deployments (cron-based sessions) and vault support for environment variable credentials.
- The Models API (`GET /v1/models`) now returns `max_input_tokens`, `max_tokens`, and a `capabilities` object per model.
- Claude Sonnet 4 and Opus 4 (original `20250514` versions) have been retired â€” all requests now return an error.
- The 1M context window beta for Sonnet 4.5 and Sonnet 4 has been retired; migrate to Sonnet 4.6 or Opus 4.6 where it's GA.

## SDK Changes

- Python SDK adds support for the new `code_execution_20260120` tool type.
- Refusal-fallback middleware requests are now tagged with `fallback-refusal-middleware` in the `x-stainless-helper` header for easier observability.
- Retired models (Sonnet 4, Opus 4 `20250514`) removed from SDK model constants.
- `APIStatusError` now includes an `error_type` field for programmatic error routing.

## Bug Fixes

- The `x-stainless-helper` header now appends values across middleware layers instead of overwriting earlier values.
- Bedrock streaming no longer drops event type information from stream events.
