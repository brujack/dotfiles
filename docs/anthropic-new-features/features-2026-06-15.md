# Anthropic & Claude API — What's New (2026-06-15)

## Model & API Changes

- **Claude Fable 5** (`claude-fable-5`) and **Claude Mythos 5** (`claude-mythos-5`) launched with 1M token context, 128k max output, and always-on adaptive thinking.
- Fable 5 and Mythos 5 use a new tokenizer that produces ~30% more tokens from the same text compared to pre-Opus 4.7 models.
- The Messages API now returns `stop_reason: "refusal"` when a safety classifier blocks a request; refused requests with no output are not billed.
- A new `fallbacks` beta parameter re-runs refused requests on another model automatically.
- `stop_details.category` on refusal responses now includes a `"reasoning_extraction"` category for reverse-engineering violations.
- Adaptive thinking is always-on for Fable 5/Mythos 5; `thinking: {"type": "disabled"}` and manual budget tokens are unsupported (return 400).
- `thinking.display` defaults to `"omitted"` on Fable 5/Mythos 5; set `display: "summarized"` to receive readable summaries.
- Fable 5 requires 30-day data retention and is unavailable under zero data retention.
- Claude Managed Agents now supports **scheduled deployments** (cron-based) and **environment variable credentials** in vaults.
- `session.thread_*` webhook events now include a `session_thread_id` field.
- The `advisor` tool gained a `max_tokens` parameter to cap output per call and reduce latency.
- The `GET /v1/environments/{id}/work` endpoint for self-hosted sandboxes is now available on Claude Platform on AWS.

## SDK Changes

- Python SDK added support for `claude-fable-5` and `claude-mythos-5` with server-side refusal fallbacks.
- Python SDK added client-side fallbacks middleware for API providers that don't support server-side fallbacks.
- Python SDK added support for Managed Agents scheduled deployments and environment variable credentials.
- Python SDK added the `frontier_llm` refusal category to the type definitions.
- Python SDK marked `claude-opus-4-1` as deprecated in type stubs.

## Bug Fixes

- Fixed Microsoft Foundry client to send the `x-api-key` header correctly for API-key authentication.
- Fixed Foundry client `copy()` and `with_options()` methods to work correctly.
- Fixed JSON schema transform to preserve `$defs` when the schema root is a `$ref`.
