# Anthropic & Claude API — What's New (2026-07-06)

## Model & API Changes
- Launched Claude Sonnet 5 with a 1M token context window, 128k max output tokens, adaptive thinking on by default, and removal of manual extended thinking and non-default sampling parameters (both now return 400 errors); it also uses a new tokenizer producing ~30% more tokens for the same text.
- Removed fast mode for Claude Opus 4.6 — requests with `speed: "fast"` now run at standard speed and pricing instead of erroring.
- Raised rate limits across the API: Sonnet and Haiku now match Opus at every tier, and usage tiers consolidated into Start, Build, and Scale.
- Deprecated fast mode for Claude Opus 4.7, with removal scheduled about a month out.
- Moved MCP tunnels management from the Admin API to the Claude API, under a new beta header and WIF scope, with the old surface still available temporarily.
- Retired the original Claude Sonnet 4 and Opus 4 models — all requests to them now error.
- Added `code_execution_20260120` support (REPL state persistence, required for programmatic tool calling) across major SDKs for Fable 5, Mythos 5, Opus 4.5+, and Sonnet 4.5+.
- Added `code_execution_20260521`, which surfaces the 90-second per-cell execution limit in the tool description.
- Added `web_search_20260318` and `web_fetch_20260318` with a `response_inclusion` parameter to drop consumed result blocks for agentic workflows.
- Managed Agents session event streams now support event deltas, backward pagination, per-session agent config overrides, and scoped vault credential injection (headers, body, or both).
- Managed Agents webhooks now cover agent, deployment, and deployment-run lifecycle events.

## SDK Changes
- Python SDK 0.116.0 adds the `agent-memory-2026-07-22` beta header.
- Python SDK 0.115.0 adds support for Managed Agents event-delta streaming, agent overrides, reverse pagination, vault credential injection scoping, and new webhook events.
- Python SDK 0.114.0 adds support for `claude-sonnet-5`.
- Python SDK 0.113.0 adds support for the 20260318 web fetch and web search tools, and now accepts user profile IDs when counting tokens.
- Removed some nonfunctional types from the SDKs in 0.115.1.

## Bug Fixes
- Fixed `agent_toolset` to allow absolute paths that resolve inside the working directory.
- Fixed the async `count_tokens` method, which was missing the `output_format`/`output_config` merge block.
