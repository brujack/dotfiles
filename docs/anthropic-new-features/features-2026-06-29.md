# Anthropic & Claude API — What's New (2026-06-29)

## Model & API Changes

- Rate limits for Claude Sonnet and Haiku now match Opus across all tiers; usage tiers consolidated to Start, Build, and Scale.
- Fast mode for Claude Opus 4.7 is deprecated, with removal July 24, 2026; migrate to Opus 4.8 fast mode.
- Claude Fable 5 (`claude-fable-5`) launched with 1M context, 128k max output, always-on adaptive thinking, and a new `stop_reason: "refusal"` for safety classifier blocks; refused requests before any output are not billed.
- Claude Fable 5 adds a `"reasoning_extraction"` refusal category to `stop_details.category` for ToS violations around model output duplication.
- Claude Fable 5 requires 30-day data retention and is unavailable under zero data retention.
- `code_execution_20260120` tool version adds REPL state persistence and is now the minimum version for programmatic tool calling; no beta header required.
- Claude Sonnet 4 and Opus 4 model aliases (`*-20250514`) are retired — all requests now return an error.
- Mid-conversation system messages are supported on Claude Opus 4.8, preserving cache hits when instructions change mid-session.
- `stop_details` field on refusal responses is now publicly documented with `category` and `explanation` fields.
- The `advisor` tool now accepts a `max_tokens` parameter to cap advisor output per call.
- Scheduled deployments and environment variable credentials are now supported in Claude Managed Agents.
- Cache diagnostics (public beta) let you pass `diagnostics.previous_message_id` to get a `cache_miss_reason` explaining prompt cache misses.
- Rate Limits API released, allowing programmatic querying of org/workspace rate limits.
- Workload Identity Federation is now GA — authenticate with short-lived OIDC tokens instead of static API keys.
- `usage.output_tokens_details.thinking_tokens` now reported in Messages API responses.
- Web search and web fetch tools support a new `response_inclusion` parameter to drop consumed result blocks from agentic workflow responses.

## SDK Changes

- Python SDK adds support for `system.message` streaming events.
- Python SDK adds the new `reasoning_extraction` refusal category and structured `stop_details` to message responses.
- Python SDK supports sending a User Profile ID in request headers.
- A Swift package (beta) adds Claude as a server-side `LanguageModel` in Apple's Foundation Models framework, callable via `LanguageModelSession`.

## Bug Fixes

- Python SDK memory tool now creates parent directories with correct permissions.
